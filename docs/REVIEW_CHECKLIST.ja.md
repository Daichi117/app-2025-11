# レビュー観点チェックリスト（FEエンジニア向け）

コードレビュー時に、まずこの順で確認してください。

## 1. 画面挙動

- 想定操作で正しく表示されるか
- ローディング表示が不自然に点滅しないか
- 空データとエラーをユーザーが区別できるか

## 2. React / Hook 依存関係

- `useEffect` の依存配列が不足していないか
- `useCallback` が古い値を閉じ込めていないか
- フォームの初期化が `effect` 乱用になっていないか

## 3. Context の責務

- その状態は本当に `context` に置くべきか
- 画面ローカル state で十分なものをグローバルにしていないか
- context 初期化時の API 失敗を吸収できているか

## 4. API 契約

- FE が送る payload と API の必須項目が一致しているか
- `response.ok` false の時に UI 側でエラー表示できるか
- `401` の時にユーザー導線が明確か

## 5. DB / Prisma

- Prisma schema の変更が migration に反映されているか
- `user_id` の条件で他人データを触れないか
- 追加テーブルやインデックスが要件に合っているか

## 6. UX

- 文言が「次に何をすればいいか」を示しているか
- ボタンが過剰で迷わないか
- モバイル幅でも主要操作が失われないか
